# 使用 commitlint 规范 commit 信息

# 安装 commitlint

安装 commitlint 和 commitlint config-conventional，然后 package.json 中配置 commitlint 以使用它。

npm install @commitlint/{cli,config-conventional} --save-dev

yarn add @commitlint/{cli,config-conventional} --dev

"commitlint": {
  "extends": ["@commitlint/config-conventional"]
}

commitlint 使用 cosmiconfigopen in new window 来支持配置文件，这意味着您还可以通过 .commitlintrc、commitlint.config.js 等文件配置 commitlint。

# 安装 husky

# Install Husky v6
npm install husky --save-dev
# Active hooks
npx husky install
# Add hook
npx husky add .husky/commit-msg 'npx --no -- commitlint --edit $1'

# Install Husky v6
yarn add husky --dev
# Active hooks
yarn husky install
# Add hook
yarn husky add .husky/commit-msg 'yarn commitlint --edit $1'

现在不按照 @commitlint/config-conventionalopen in new window 规范提交的 commit 会被阻止并给出错误提示：

$ git commit -m 'project init'
